# CSAgent 开发计划

## 1. 当前阶段定义

当前 `CSAgent` 已完成的是 **Phase 0：框架骨架初始化**。

该阶段的目标不是全量替换旧版 `telecom-agent`，而是先建立一个适合长期演进的新主目录。

当前 Phase 0 已覆盖：

- 新仓库目录
- 后端入口
- 前端通用聊天壳
- 插件目录
- 兼容桥接模式
- 迁移说明与标准文档

## 2. 后续开发总路线

建议总共按 5 个阶段推进。

## Phase 1：把框架核心做实

### 目标

把当前只是“能承载”的结构，升级为“真的有框架能力”的结构。

### 要做的事

- 实现正式的 Tool Registry
- 实现正式的 Skill Loader
- 实现 Card Schema Registry
- 实现 Plugin Loader 和依赖校验
- 实现 Provider 抽象
- 实现 Agent Runtime 主循环

### 交付物

- `backend/app/runtime/registry.py`
- `backend/app/runtime/provider.py`
- `backend/app/runtime/agent_loop.py`
- `backend/app/plugins/loader.py`

### 完成标志

- `native_core` 可以不依赖旧版桥接完成一轮最小聊天流程
- Tool / Skill / Card 元数据可以被框架读取并参与运行

## Phase 2：迁移全局能力

### 目标

把不依赖具体业务域的能力迁入新框架。

### 优先迁移项

- `load_skills`
- `search_knowledge` 的协议层
- 会话状态对象
- 工作流状态对象
- Tool Policy 规则

### 完成标志

- 全局工具可以在 `CSAgent` 原生运行
- 技能切换不再依赖旧版仓库

## Phase 3：迁移低风险业务插件

### 目标

先迁移查询与推荐这两类相对清晰、风险较低的能力。

### 迁移顺序

1. `telecom.query`
2. `telecom.recommend`

### 每个插件都要做的事

- 迁移 Tool 参数 schema
- 迁移 Tool 结果 schema
- 迁移 Card schema
- 迁移 Skill prompt
- 迁移前端卡片组件
- 增加最小测试

### 完成标志

- 查询和推荐场景可以完全在 `native_core` 下运行
- 前端可以渲染查询卡和推荐卡

## Phase 4：迁移高风险业务插件

### 目标

迁移订单与充值能力。

### 迁移顺序

1. `telecom.recharge`
2. `telecom.order`

### 原因

- 充值相对简单，主要是金额编辑与链接生成
- 下单涉及限制校验、验证码、支付确认和工作流约束，复杂度最高

### 完成标志

- 充值链路原生可用
- 下单和支付确认链路原生可用
- `legacy_bridge` 不再承担主链路

## Phase 5：去桥接与平台化

### 目标

把 `CSAgent` 变成独立主仓库，并开始考虑平台级能力。

### 要做的事

- 去掉旧版桥接依赖
- 数据库与迁移体系标准化
- 认证与限流
- 插件脚手架生成器
- 插件测试基线
- 可观测性与回放

## 3. 最近两周建议执行项

下面是最值得你优先做的 backlog。

### 第一优先级

- 实现正式 Plugin Loader
- 实现 Tool 自动发现与注册
- 扩展 Skill frontmatter 结构
- 定义统一 Card schema 顶层字段

### 第二优先级

- 迁移 `load_skills`
- 迁移查询插件
- 给前端加 Card Renderer 注册表
- 给聊天 runtime 加统一 Event 模型

### 第三优先级

- 迁移推荐插件
- 抽离 workflow state
- 给动作类 Tool 补幂等、确认和冲突规则

## 4. 每阶段验收标准

### 架构验收

- 新增能力必须进入 `plugins/`，而不是散落写在核心目录
- 核心目录不能混入电信专属业务实现

### 运行验收

- `legacy_bridge` 模式可用
- `native_core` 模式逐步补齐并可跑通场景

### 文档验收

- 每个插件必须有 README
- 每个 Skill 必须有 `SKILL.md`
- 每个 Tool 必须有稳定 schema

## 5. 你现在可以直接继续做什么

如果你下一步继续在这个目录里开发，最推荐先做下面三件事：

1. 把 Plugin Loader 从“扫 manifest”扩展成“扫 manifest + 扫 tool + 扫 skill”
2. 把 `native_core` 从当前最小响应升级成真正的 Agent Runtime
3. 从 `telecom.query` 开始做第一批真实迁移
